# How Websites Work

Learned that when visiting a website:

1. Browser sends a request to a web server
2. Server processes the request
3. Server sends back data
4. Browser renders the webpage

## Key Concepts
- Client = browser/user device
- Server = remote computer hosting website
- Front End = visible part of website
- Back End = server-side processing and databases

---

# HTML Fundamentals

HTML is used to structure webpages.

## Basic HTML Structure

```html
<!DOCTYPE html>
<html>
<head>
    <title>Page Title</title>
</head>

<body>
    <h1>Main Heading</h1>
    <p>Paragraph text</p>
</body>
</html>
```

## Important Tags Learned
- `<html>`
- `<head>`
- `<body>`
- `<h1>`
- `<p>`
- `<img>`
- `<button>`

# Attributes

Attributes provide extra information to HTML elements.

Example:

```html
<p class="red-text">Hello</p>
```

## Important Attributes
- class
- id
- src
- href

---

# Class vs ID

## class
- Used for grouping multiple elements
- Commonly used for styling

## id
- Unique identifier for a single element
- Used by CSS and JavaScript

---

# Cybersecurity Notes

- HTML source code is visible to users
- Users can inspect and modify front-end code using Developer Tools
- Client-side code should never be trusted for security
- Browser DevTools are important for web analysis and security testing

---

# Main Takeaway

Websites are built using structured components:
- HTML for structure
- CSS for styling
- JavaScript for interactivity

Understanding website structure is essential for cybersecurity and web application security.

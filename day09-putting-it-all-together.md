# Day 9 - Putting It All Together

## What I Learned

- When a user requests a website, the browser uses DNS to find the server's IP address.
- The browser communicates with the web server using HTTP/HTTPS.
- A web server serves website files from a root directory, such as `/var/www/html` on Linux.
- Load balancers distribute traffic across multiple servers and help with failover.
- CDNs serve static files like images, CSS, JavaScript, and videos from servers closer to the user.
- Databases store and retrieve user/application data.
- A WAF protects web applications by filtering suspicious or malicious requests.
- Virtual hosts allow one web server to host multiple websites using the Host header.
- Static content is served directly and usually does not change.
- Dynamic content is generated by backend code and may change depending on user input.

## Key Terms

- DNS: Translates domain names into IP addresses.
- Web Server: Software that receives HTTP requests and sends back web content.
- Root Directory: Main folder where website files are stored.
- Virtual Host: Configuration that allows one server to host multiple websites.
- Load Balancer: Distributes requests across multiple servers.
- CDN: Network of servers that deliver static content faster.
- Database: Stores application and user data.
- WAF: Web Application Firewall that protects web apps from attacks.
- Frontend: What the user sees in the browser.
- Backend: Server-side logic that processes data and generates responses.

## Main Takeaway

A website is not just one page or one server. Behind the scenes, DNS, web servers, load balancers, CDNs, databases, backend code, and security tools like WAFs all work together to deliver the final webpage to the browser.
